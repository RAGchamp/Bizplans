1. Executive Summary

Every large bank, insurer, airline, and government agency runs mission-critical logic on COBOL, IMS, and DB2 systems that are decades old. The people who understand this code are retiring, documentation is thin or missing, and every prior "just rewrite it" attempt has been slow, expensive, and high-risk. Meanwhile a new generation of agentic coding tools — able to plan, edit files, run commands and tests, and return a reviewable diff — has matured enough to meaningfully help, but no one has packaged that capability specifically for legacy modernization, and few enterprises will point an ungoverned AI agent at a core banking system.

LegacyClaw is an open source Python framework that combines two existing building blocks — LangChain's Deep Agents Code (dcode) harness and NVIDIA NemoClaw, a governed blueprint that runs Deep Agents Code on NVIDIA Nemotron 3 Ultra inside an NVIDIA OpenShell runtime — into a purpose-built pipeline for legacy system transformation. It does not reinvent the agent or the model; it is the domain layer that turns a general-purpose, governed coding agent into a repeatable legacy modernization workflow: repository ingestion, COBOL/IMS/DB2-aware planning, coordinated multi-file transformation to Java, automated test generation and execution, and a reviewable diff — all inside a deny-by-default, audited runtime.

MVP scope: convert a COBOL / IMS / DB2 based system into a working Java-based system, end to end, as a downloadable, runnable, open source Python project.

This plan has two engines, working together:

  1. Adoption engine (the open source framework): the free, downloadable MVP — target 10,000 downloads in Year 1 — builds credibility, proves the LangChain Deep Agents Code + NVIDIA NemoClaw combination works on a genuinely hard problem, and generates the inbound enterprise interest that feeds the second engine.
  2. Revenue engine (Human-in-the-Loop services): every LegacyClaw migration ends with a mandatory human sign-off step (see Framework Architecture, Stage 5 — Review). LegacyClaw monetizes that exact step from Year 1 by supplying vetted, experienced COBOL/Mainframe and Java engineers as Human-in-the-Loop (HITL) reviewers, billed to clients at $100/hour, paid out to the resources at $70/hour, retaining $30/hour as margin.
Assuming 160 billable hours/resource/month, the first resource engaged in Month 6, and a ramp to 50 cumulative resource-months (8,000 hours) by year end: $800,000 gross client billings → $560,000 paid to resources → $240,000 net margin revenue retained by LegacyClaw.

Against $35,000 in total Year-1 operating costs (infrastructure $10,000; sales, marketing & operations $20,000; setup & legal $5,000), this plan lands at an estimated $205,000 pre-tax profit in Year 1 — and roughly $143,500 after an assumed ~30% tax rate — funded entirely by the HITL services layer, while the open source framework itself stays free (see Human-in-the-Loop Services Model and Cost Structure & Profit Model).

Year 2+ expands on both engines: broadening supported technology (e.g., data warehouse modernization from T-SQL to Snowflake, in addition to COBOL/IMS/DB2 → Java), scaling the HITL staffing model across that wider surface, and adding a second monetization channel — qualified lead generation for AI infrastructure/tooling companies (e.g., LangChain) — on top of continued HITL services (see Business Model).

2. The Problem

  • The knowledge is disappearing faster than the code. COBOL, IMS, and DB2 systems were built by a generation of engineers who are now retiring; in many organizations the number of people who can safely modify a given legacy module is a single digit — or zero.
  • The systems are too important to leave alone and too risky to touch by hand. These systems run core banking, insurance claims, airline reservations, and government benefits processing. A hand-written rewrite carries years of timeline risk and a real chance of subtle business-logic regressions that surface only in production.
  • Prior modernization approaches have earned skepticism. Rules-based transpilers produce Java that compiles but is unreadable and unmaintainable ("COBOL in a Java suit"). Big-consultancy manual rewrite programs are measured in years and tens of millions of dollars, with uneven success rates. Neither builds lasting internal capability.
  • General-purpose AI coding agents are improving fast, but enterprises can't just point one at core systems. The same plan-edit-test-review loop that makes agentic coding compelling for greenfield work is exactly what's needed for legacy migration — but without deny-by-default networking, approval controls, and an audit trail, no risk-averse enterprise will allow an autonomous agent near a COBOL/DB2 production codebase.
  • Even a governed agent still needs a qualified human to sign off. No enterprise will merge agent-generated changes to a core system without an experienced COBOL/Mainframe and Java engineer reviewing the transformation — but most enterprises don't have spare senior engineers sitting around for this, which is exactly the resourcing gap LegacyClaw's Human-in-the-Loop layer fills.
  • No one has packaged the agentic approach specifically for legacy modernization. Deep Agents Code is a general-purpose terminal coding agent; NemoClaw is a general-purpose governed runtime blueprint. Neither, out of the box, understands COBOL copybooks, IMS segments/PSBs, DB2 schemas and embedded SQL, or the shape of a COBOL-to-Java transformation plan. That domain layer is the gap LegacyClaw fills.

3. Technical Foundation — What LangChain Deep Agents Code + NVIDIA NemoClaw Already Provide

LegacyClaw is not built from scratch — it is an application layer on top of two components that already exist and already work.

ComponentWhat it already provides
LangChain Deep Agents Code (dcode)An open source terminal coding agent built on the Deep Agents SDK: task planning, repository inspection, coordinated multi-file edits, command/test execution, iterative review, and diff output. Manages long-running context through summarization, result offloading, persistent memory, skills, and subagents.
NVIDIA NemoClawA governed blueprint that runs Deep Agents Code on NVIDIA Nemotron 3 Ultra inside an NVIDIA OpenShell runtime: deny-by-default networking, approval controls, and a full audit trail — so the agent can work on sensitive codebases while source code, credentials, execution, and network access stay under enterprise control.
Open, model-neutral architectureBecause the harness (Deep Agents Code) and the model (Nemotron 3 Ultra) are cleanly separated, teams can adapt harness, model, and runtime independently as needs change — LegacyClaw inherits this flexibility rather than locking users into one vendor's closed stack.

What this proves: the hardest infrastructure problems — a capable coding agent loop, and a governed runtime an enterprise can actually trust with production source code — are already solved. LegacyClaw's job is the domain-specific layer on top: teaching this general-purpose, governed agent how to actually read and transform COBOL/IMS/DB2 systems into Java, packaged as a reusable framework rather than a one-off consulting engagement.

What LegacyClaw adds that doesn't exist yet:

  • COBOL/IMS/DB2-aware repository ingestion (copybooks, IMS DBDs/PSBs, DB2 DDL and embedded SQL, JCL job structure) into a structured model the agent can plan against.
  • A legacy-to-Java migration playbook encoded as Deep Agents skills and persistent memory — conventions, transformation patterns, and review standards the agent reuses across files and runs.
  • Subagents specialized by concern: data-layer transformation (IMS/DB2 → JPA/JDBC), business-logic transformation (COBOL procedure division → Java service classes), and test-equivalence verification.
  • A packaged, pip install-able Python framework with a CLI, example legacy codebases, and a documented workflow — value a developer can experience in an afternoon, not something that needs a sales conversation first.
  • A Human-in-the-Loop review network — the vetted COBOL/Mainframe and Java engineers who staff the mandatory Stage 5 sign-off (see Framework Architecture), which is where the business actually monetizes.

4. The Product — LegacyClaw Framework (MVP Scope)

MVP definition: given a COBOL / IMS / DB2 based system (a bundled example system plus support for bring-your-own-codebase), LegacyClaw produces a working, compiling, tested Java-based system, along with a reviewable transformation plan and diff — running end-to-end inside the NemoClaw/OpenShell governed environment.

In scope for MVP
  • Ingestion of COBOL source, copybooks, IMS DBD/PSB definitions, DB2 DDL, and embedded SQL/CICS calls into an internal representation.
  • An agent-generated migration plan: module inventory, dependency graph, proposed Java package/class structure, sequenced transformation order.
  • Coordinated code transformation: COBOL programs → Java classes/services, IMS/DB2 data access → a chosen Java persistence approach (e.g., JPA entities + repository layer), preserving business logic and control flow.
  • Automated test generation and execution: equivalence tests from sample legacy I/O run against the produced Java, flagging behavioral drift.
  • A final reviewable diff and migration report a human engineer signs off on before anything is merged — the agent proposes, a human disposes. For paying clients this sign-off is fulfilled by LegacyClaw's own Human-in-the-Loop resources (see Section 8).
  • Runs entirely on the LangChain Deep Agents Code + NemoClaw stack, so governance is real from day one.
Explicitly out of scope for MVP
  • Other legacy source languages/platforms (PL/I, Natural/Adabas, RPG, mainframe Assembler) — the architecture is designed to generalize, but the MVP proves the concept on one pair: COBOL/IMS/DB2 → Java.
  • Data warehouse/ETL modernization (e.g., T-SQL → Snowflake) — planned as a Year 2 technology expansion (see Business Model), not a Year 1 dependency.
  • Automated production cutover/deployment tooling — the MVP's output is a reviewed Java codebase and test suite, not a live cutover.
  • A hosted/managed version of the service — MVP is a self-hosted, downloadable framework only.
  • UI/dashboard beyond the CLI and generated reports.

These are candidates for the post-MVP roadmap once the core loop is proven — see Build Plan & Roadmap.

Primary (MVP — download/adoption audience, and the buyer of HITL services)
  • Enterprise architects and modernization leads at banks, insurers, airlines, and government agencies evaluating agentic approaches to mainframe modernization — the same organizations that download the free framework are the natural buyers of vetted human reviewers to sign off on it.

7. Business Model — Open Source + Human-in-the-Loop Services

LegacyClaw runs two engines in parallel from Year 1, not in sequence — the open source tool is not expected to fund itself through sales; a services layer built on top of it is.

Engine 1 Open core, free (adoption)

MIT/Apache-licensed. The full MVP pipeline (ingest → plan → transform → verify → review) is free, downloadable, and runnable by anyone with access to NemoClaw/Nemotron. Its job is credibility and top-of-funnel reach — the 10,000-download goal (see Growth Model) — and, critically, generating the inbound enterprise relationships that become HITL clients.

Engine 2 Human-in-the-Loop services (Year 1 revenue)

A bench of experienced, vetted COBOL/Mainframe and Java engineers who work solely as Human-in-the-Loop reviewers/approvers on LegacyClaw-assisted migrations.

Client bill rate$100/hr
Resource pay rate$70/hr
Margin retained by LegacyClaw$30/hr

This is a staffing/margin business, not a headcount business — LegacyClaw does not need its own migration engineers; it needs a vetted bench, a sales motion to place them against LegacyClaw-run migrations, and the open source tool's credibility to get in the door. Full ramp and unit economics are in Section 8.

Year 2+ Expand both engines
  • Technology expansion — broaden supported source/target pairs beyond COBOL/IMS/DB2 → Java, starting with data warehouse/ETL modernization (e.g., T-SQL → Snowflake), and scale the HITL staffing model across that wider surface.
  • New monetization channel — AI company lead generation — every HITL engagement puts LegacyClaw in direct contact with an enterprise actively buying agentic AI tooling for a real, budgeted project; that qualified buying intent is monetizable as a referral/lead-gen channel for AI infrastructure/tooling companies (e.g., LangChain), on top of continued HITL revenue.
  • Deeper co-marketing with NVIDIA and LangChain, now backed by a proven Year-1 revenue track record and real client case studies, not just a download count.

9. Market Sizing (Directional)

  • Mainframe-resident COBOL code in production today is commonly estimated in the hundreds of billions of lines, concentrated in banking, insurance, government, and airline systems — a market that isn't shrinking as fast as headcount able to maintain it.
  • Legacy modernization services is an existing, well-funded enterprise IT spending category (measured in tens of billions of dollars annually across consultancies and system integrators) — LegacyClaw doesn't need to create this market, it needs to give existing buyers and sellers a more credible AI-accelerated toolchain plus the specialist staffing to review its output.
  • Experienced COBOL/Mainframe engineers are a shrinking, aging, and increasingly expensive labor pool — precisely the supply-side constraint the HITL staffing model is priced against, and the reason enterprises will pay a margin for LegacyClaw to source and vet that talent rather than recruiting it themselves.
  • Developer/AI-engineering audience curious about agentic coding on hard problems is large and growing quickly — millions of developers actively exploring coding agents, LLM harnesses, and enterprise AI blueprints — the more immediately reachable audience for the 10,000-download Year 1 goal, and the top of the funnel that surfaces HITL buyers.
  • COBOL/mainframe skills shortage is a widely cited, recurring enterprise pain point — a structural tailwind that doesn't depend on LegacyClaw's own marketing to sustain interest.

12. Build Plan & Roadmap

Because the coding agent harness (Deep Agents Code) and the governed runtime (NemoClaw/Nemotron 3 Ultra/OpenShell) already exist, Year 1 build effort is concentrated on the COBOL/IMS/DB2 domain layer, the migration playbook, packaging/distribution, and standing up the HITL staffing operation — not on rebuilding agent infrastructure.

Phase 1 — Domain Ingestion & Repo Model (Months 1–2)
  • COBOL, copybook, IMS DBD/PSB, and DB2 DDL/embedded-SQL parsers producing a structured repository model
  • A bundled example legacy system (representative COBOL/IMS/DB2 sample application) for demos and onboarding
  • Initial Deep Agents Code skills/memory encoding basic migration conventions
Phase 2 — Planning & Transformation Pipeline (Months 2–5)
  • Migration planning agent: dependency graph, target Java structure, sequencing
  • Business-logic transformation subagent (COBOL procedure division → Java services)
  • Data-layer transformation subagent (IMS/DB2 access → Java persistence layer)
  • Integration with NemoClaw/OpenShell for governed execution (approvals, audit trail, deny-by-default networking)
  • Begin recruiting and vetting the first cohort of Human-in-the-Loop COBOL/Mainframe and Java reviewers, targeting the Month 6 first-engagement milestone
Phase 3 — Verification & Public MVP Launch (Months 5–7)
  • Equivalence test generation from sample legacy I/O and automated test execution
  • Migration report/diff generation for human review
  • Packaging (pip install legacyclaw, CLI, docs, quickstart with the bundled example system)
  • Public launch: GitHub, Product Hunt, Hacker News, NVIDIA/LangChain community channels
  • First Human-in-the-Loop resources engaged (Month 6) and first billable client engagement(s)
Phase 4 — Community Growth, Playbook Expansion & HITL Ramp (Months 7–12)
  • Distribution scaling on best-performing channels (see Go-to-Market)
  • Community-contributed skills/playbooks for additional migration patterns within COBOL/IMS/DB2 → Java scope
  • HITL bench ramp from 2 to 10 active resources (see Section 8)
  • First enterprise pilot engagement(s), feeding real-world case study content back into distribution
  • Early scoping of Year 2 technology expansion (T-SQL → Snowflake) and AI-company lead-generation channel

14. Competitive Landscape

  • Traditional COBOL-to-Java transpilers (rules-based, vendor tools) — produce compiling but often unmaintainable "COBOL in a Java suit" output; no planning, review, or test-equivalence loop, and no staffing layer. LegacyClaw's agentic pipeline targets genuinely maintainable, idiomatic Java, with a human-reviewable diff — and the vetted human to review it.
  • Big-consultancy manual rewrite programs — capable of high-quality outcomes but slow and expensive, measured in years and tens of millions of dollars; LegacyClaw is a credible accelerator these same consultancies can adopt, and a potential staffing partner, not a head-on competitor for the relationship.
  • Traditional COBOL/mainframe staffing agencies — supply similar talent but with no accompanying agentic tooling, no governed runtime, and no free-tool-driven lead generation; LegacyClaw's staffing margin is defensible because the placement comes bundled with the tool that created the client relationship in the first place.
  • General-purpose AI coding agents (Devin, GitHub Copilot workspace/agent modes, OpenAI Codex-based agents, other Deep Agents SDK applications) — increasingly capable at general software engineering, but none are packaged with a COBOL/IMS/DB2-aware domain layer, a governed runtime, or a staffing model to fill the human-review gap.
  • Other NVIDIA NemoClaw blueprints / LangChain Deep Agents Code applications — as the ecosystem grows, other domain-specific applications will emerge; LegacyClaw's differentiation is being first and most credible in the legacy-modernization niche, with a working MVP, a community, and — uniquely — a revenue model that doesn't depend on the tool itself being paid.

Differentiation: LegacyClaw is the first packaged, open source, purpose-built application of a governed agentic coding harness to COBOL/IMS/DB2 modernization that monetizes the human-review step itself rather than the software — combining the trust story (NemoClaw's governance), the domain story (a real migration playbook), and a staffing business model that turns the tool's own governance requirement into the revenue engine.

17. Success Metrics

  • HITL net margin revenue — target: $240,000 in Year 1 (8,000 billable hours at $30/hr margin)
  • HITL bench utilization — active resources vs. billable hours actually delivered against the 160 hrs/resource/month assumption
  • Total downloads/installs (PyPI + GitHub releases + container pulls) — target: 10,000 in Year 1
  • GitHub stars, forks, and external contributors — leading indicator of genuine community adoption
  • Migration correctness rate on the bundled example system and any client codebases (equivalence test pass rate) — the core trust metric underpinning both the free tool and the paid review service
  • Conversion rate from open-source download/inbound interest to a signed HITL engagement — the key linkage metric between the two engines
  • Time from pip install to a completed example migration — a proxy for how well the framework delivers on the "showcase the power of NemoClaw and LangChain" promise in a single sitting
  • Resource retention/attrition on the HITL bench — leading indicator for Year 2 staffing scalability
  • NVIDIA/LangChain co-marketing placements secured (blueprint marketplace listing, joint blog posts, conference features) — proxy for ecosystem-channel distribution health
